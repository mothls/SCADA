SCADA ДЛЯ ВЕТРО-СОЛНЕЧНОЙ УСТАНОВКИ (УЧЕБНЫЙ СТЕНД)

Разработано в рамках дипломного проекта.
Система собирает данные с оборудования (Modbus RTU, MQTT), рассчитывает SOC батареи, применяет защитный алгоритм и отображает всё в графическом интерфейсе.
Поддерживает режим симуляции для работы без реального железа.

ОПИСАНИЕ ФАЙЛОВ

Корень проекта
- .gitignore – исключает из Git: виртуальное окружение, кэш, файлы БД, конфиги с паролями.
- config.ini.example – пример конфигурации. Скопируйте в config.ini и отредактируйте под своё оборудование.
- requirements.txt – зависимости, устанавливаемые через pip install -r requirements.txt.
- README.md – документация.

src/
main.py
Запускает приложение:
- загружает конфигурацию,
- инициализирует базу данных,
- создаёт клиентов Modbus/MQTT (или симуляцию, если SIMULATION_MODE = True),
- запускает ControlLoop (1 Гц) в отдельном потоке,
- создаёт и показывает главное окно,
- обрабатывает завершение программы (остановка цикла, закрытие соединений).

src/core/
state_manager.py
Единый потокобезопасный кэш состояния системы.
- Хранит все параметры устройств (напряжение, ток, температура, SOC, RPM и др.)
- Управляет флагами безопасности (_safety_flags)
- Управляет отложенными командами (_pending_commands)
- Методы: get(), update(), set_safety_flag(), get_safety_flag(), get_pending(), clear_pending()

database.py
Модуль для работы с SQLite (через sqlite3).
- Класс Database с методами:
  - add_device_data() – сохранение данных устройств (vin, vout, iin, iout, temp)
  - add_telemetry() – сохранение произвольных параметров (RPM, углы)
  - add_event() – запись системных событий и аварий
  - get_latest_data() – получение последней записи по устройству
  - get_history_for_param() – получение истории по параметру
- Таблицы создаются автоматически при инициализации.
- Генерирует сигнал data_added для обновления графиков.

modbus_client.py
Клиент для работы с Modbus RTU.
- Подключение к COM-портам (солнечный конвертер: COM3, ветровой: COM4)
- Чтение регистров: напряжение, ток, температура
- Управление выходом: output_enable(True/False)
- Установка уставок: set_voltage_and_current(v, i)
- Интеграция с датчиком Холла (COM5) для измерения тока АКБ

mqtt_client.py
Клиент MQTT для связи с ветрогенератором и нагрузкой.
- Подключение к брокеру (127.0.0.1:1883)
- Подписка на топики:
  - stat/lab/plug/POWER – статус нагрузки
  - lab/wind_turbine/rpm – обороты ветрогенератора
  - lab/wind_turbine/brake_cmd – состояние балласта
  - lab/wind_turbine/status – статус ESP32
- Публикация команд:
  - cmnd/lab/plug/POWER -> ON/OFF
  - cmnd/lab/wind_turbine/brake_cmd -> BRAKE_ON/BRAKE_OFF

control_loop.py
Главный цикл управления (1 Гц), выполняется в отдельном потоке.
Последовательность такта:
1. _gather_state() – опрос всех устройств (Modbus, MQTT, датчик Холла)
2. _calculate_load_current() – расчёт тока нагрузки по закону Кирхгофа
3. _update_battery_soc() – расчёт SOC батареи методом кулонометрии
4. _check_safety() – вызов SafetyShield (защитный алгоритм)
5. _run_student_script() – выполнение пользовательского скрипта
6. _arbitrator.arbitrate() – арбитраж команд по приоритетам
7. _execute() – отправка финальных команд на устройства
8. time.sleep(1.0) – пауза до следующего такта

safety_shield.py
Защитный алгоритм с высшим приоритетом.
- Возвращает принудительные команды для арбитратора
- Сценарии:
  - Ветер: RPM >= 1200 -> 3 попытки изменения угла атаки; RPM >= 1500 -> балласт + отключение конвертера; RPM >= 1800 -> EMERGENCY
  - Перегрев: T >= 65C -> снижение тока на 20%; T >= 70C -> отключение конвертера
  - АКБ: SOC >= 90% -> включить нагрузку, отключить конвертеры; SOC <= 10% -> отключить нагрузку, включить заряд (14.5В, 5А)
  - Перегрузка: |I_bat| > 20А -> EMERGENCY
- Методы: evaluate() -> dict команд, reset_emergency() – ручной сброс

command_arbitrator.py
Формирует финальные команды с учётом приоритетов:
1. EMERGENCY – полная остановка
2. SafetyShield – защитные команды
3. GUI – ручной режим оператора
4. Скрипт – пользовательский алгоритм (не может переопределить защиту)

api_wrapper.py
Программный интерфейс для пользовательских скриптов.
- Sensors – чтение данных (напряжение, ток, SOC, RPM, температура и др.)
- Actuators – управление (нагрузка, конвертеры, балласт, уставки)
- System – логирование

sandbox_runner.py
Изолированное выполнение пользовательских скриптов.
- Ограничивает доступ к опасным функциям (os, sys, socket, serial)
- Выполняет функцию loop(api) каждый такт
- Перехватывает исключения, не ломая основной цикл

simulation.py
Режим симуляции (без реального железа).
- MockConverter – виртуальный конвертер (синусоида 24В, ток 2-6А)
- MockMQTT – виртуальный MQTT (RPM 800+/-300, нагрузка переключается каждые 15 сек)
- Полная совместимость с реальным API
- Переключатель в main.py: SIMULATION_MODE = True/False

src/gui/
main_window.py
Главное окно на PyQt5.
- Меню: Настройки, Алгоритмы, Справка
- Статусная строка с индикаторами подключения, балласта, АКБ, нагрузки, защиты
- Интеграция всех компонентов GUI

mnemonic_widget.py
Интерактивная мнемосхема.
- Загружает изображение из resources/mnemonic_shem.jpg
- 5 интерактивных зон: солнечный конвертер, ветровой конвертер, нагрузка, ветрогенератор, АКБ
- При клике по зоне – popup с параметрами (SOC, ток, напряжение, статус)
- Цветовая индикация статусов

charts_area.py
Область управления графиками (правая часть окна).
- Содержит до 4 виджетов ChartWidget
- Динамически перестраивает компоновку (1 график – вертикально, 2 – сплиттер, 4 – сетка 2x2)
- Предотвращает добавление дублирующихся графиков

chart_widget.py
Виджет графика (на основе QChart).
- Отображает линию последних 100 измерений
- Оси: X – время (ЧЧ:ММ:СС), Y – значение (ток/напряжение/температура/RPM)
- Tooltip при наведении на точку
- Поддержка масштабирования прямоугольной областью
- Автоматически обновляется при добавлении новой записи в БД

algorithm_manager.py
Виджет управления пользовательскими скриптами (правая панель главного окна).
- Кнопки управления:
  - "Загрузить скрипт" – выбор Python-файла через диалог (QFileDialog)
  - "Запустить" – загрузка скрипта в SandboxRunner и старт выполнения
  - "Остановить" – остановка скрипта и сброс состояния
- Текстовая область "Логи выполнения скрипта..." – отображение stdout/stderr от скрипта в реальном времени
- Таймер обновления статуса (1 Гц) – проверяет состояние песочницы и выводит ошибки
- Блокировка кнопок:
  - После загрузки скрипта кнопка "Загрузить" блокируется
  - После запуска блокируются "Запустить" и "Загрузить", активируется "Остановить"
  - После остановки все кнопки разблокируются

src/utils/
config.py
Загружает конфигурацию из config.ini.
- Чтение секций: [Modbus], [MQTT], [Database], [Safety], [Battery]
- Значения по умолчанию (fallback)
- Возвращает объект ConfigParser

УСТАНОВКА И ЗАПУСК

1. Установите Python 3.8.
2. Клонируйте репозиторий (или скопируйте файлы).
3. Создайте виртуальное окружение и активируйте его:
   python -m venv venv
   venv\Scripts\activate
4. Установите зависимости:
   pip install -r requirements.txt
5. Скопируйте config.ini.example в config.ini и отредактируйте под своё оборудование.
6. Запустите приложение:
   cd src
   python main.py

РЕЖИМ СИМУЛЯЦИИ

Для работы без реального железа установите в main.py:
SIMULATION_MODE = True
Система будет использовать виртуальные данные (синусоиды, случайные числа).
Для работы с реальным оборудованием:
SIMULATION_MODE = False

ПОЛЬЗОВАТЕЛЬСКИЕ СКРИПТЫ

Пример скрипта для песочницы:
def loop(api):
    soc = api.sensors.get_battery_soc()
    if soc < 20:
        api.actuators.turn_load_off()
    elif soc > 80:
        api.actuators.turn_load_on()

Скрипт загружается через меню "Алгоритмы -> Загрузить скрипт".
Защитный алгоритм имеет приоритет над скриптом и не позволяет нарушить безопасные режимы работы.
